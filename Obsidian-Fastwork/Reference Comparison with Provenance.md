# Reference Comparison with Provenance

การศึกษาระบบอื่นมีประโยชน์เมื่อแยก “ของเขา/ของเรา” และผูก claim กับ source ที่ตรวจได้ ไม่ใช่คัด feature list

- บันทึก commit/version และ file:line หรือเอกสารต้นทาง
- เปรียบเทียบ gap → mechanism → เงื่อนไขที่เหมาะกับ T4
- ระบุ unknown และวิธีตรวจ เช่น read-code, run-experiment, ask-owner
- จดสิ่งที่พิสูจน์แล้วว่าไม่ใช่สาเหตุ เพื่อลดการวนซ้ำ
- ตรวจ upstream ปัจจุบันก่อนพึ่ง claim เก่า และอย่า sync vendor แบบตาบอด
